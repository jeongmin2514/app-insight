# 미니앱 3종 그로스 대시보드 (app-insight) — 디자인 스펙

> 작성 2026-07-11. 브레인스토밍 확정본.
> 목적: 취업 포트폴리오 자산 — 데이터분석가·데이터마케터·그로스분석가 3직무 공용.

## 1. 목표와 서사

- 한 줄 정체성: **"지표를 직접 정의하고, 수집 파이프라인을 만들고, 해석해서 다음 액션으로 연결한 사람"의 물증.**
- 산출물 2개: 공개 웹 대시보드 링크 1개(Vercel) + public GitHub 레포 1개.
- 기존 마스터서류 확정 서사("분석만 하는 게 아니라 직접 만들어서 검증하는 사람")를 뒷받침한다. 기존 대표 숫자(구름한입 92% / 잡플로우 1위)와 경쟁하지 않고 보강한다.

### 직무별 인용 매핑 (대시보드 1개 → 서류 3용도)

| 대시보드 섹션 | 어필 역량 | 인용하는 서류 버전 |
|---|---|---|
| 파이프라인 + 지표 정의 (Python ETL, DAU·리텐션 산식) | 데이터 핸들링·지표 설계 | 데이터분석가 |
| 푸시 캠페인 성과 (세그먼트별 클릭률, 리타겟 vs 신규) | 캠페인 운영·타겟팅 | 데이터마케터 |
| 리텐션·퍼널 + 인사이트 노트 (해석→액션→재측정) | 실험·그로스 루프 | 그로스분석가 |

## 2. 확정된 요구사항

- 형태: 웹 라이브 대시보드 1차 → 같은 데이터(CSV)로 Tableau Public 2차.
- 수집: **반자동 스냅샷** — 주 1~2회 로컬 세션에서 토스 콘솔 MCP 조회 → 적재 → git push → Vercel 자동 배포. 완전 무인 자동화는 하지 않는다(콘솔 인증이 로컬 계정 단위).
- 콘텐츠: 지표 차트 + **인사이트 노트**(관찰 → 해석 → 액션) 결합.
- 공개 범위: 실수치 **전부 공개** (DAU·리텐션·푸시 성과·수익 포함. 개인 프로젝트라 NDA 이슈 없음).

## 3. 아키텍처 — 정적 ETL 파이프라인

```
[Extract]  토스 콘솔 MCP 조회 (주 1~2회, 로컬 클로드 세션)
              → data/raw/{app}/{metric}-{범위}.json   ← 원본 그대로 보존
[Transform] scripts/build_dashboard.py (Python·pandas)
              → data/dashboard.json   (웹용 단일 파일: 지표 + 노트 임베드)
              → data/csv/*.csv        (Tableau 2차용)
[Load/Viz]  index.html + Chart.js (CDN) → git push → Vercel 정적 배포
[Notes]     notes/YYYY-MM-DD-<slug>.md → 빌드 시 dashboard.json에 포함 → 타임라인 렌더
```

기술 선택 이유(README에도 기재):
- 서버·DB 없음 — 반자동 스냅샷 구조에 정적 호스팅이면 충분. 인프라 비용 0.
- 가공은 Python(pandas) — 직무 서사(Extract는 콘솔 API, Transform은 Python, Visualize는 웹·Tableau)와 일치.
- Next.js 등 프레임워크 비채택 — 개발자 전향 인상 방지(서류작성_필수기준 §2와 일치), 분석 직무엔 과투자.

### 수집 대상 지표

- 앱 3종(구름한입·잡플로우·데일리픽) 공통: DAU, 세션, 리텐션, 전환, 페이지뷰.
- 구름한입 추가: 푸시 성과(`push_stats` — 세그먼트별 발송·클릭), 광고 수익(선택).
- 콘솔 MCP는 계정 단위 인증 → **수집 시 대상 앱 특정 필수** (세 앱 데이터 섞임 방지).

### 적재 규칙

- raw 스냅샷은 날짜 키 기준 **upsert 병합**: 같은 날짜를 재수집하면 최신값으로 덮어쓰고, 새 날짜는 추가. 재실행해도 안전(멱등).
- 원본(raw)은 절대 수정하지 않는다. 가공 로직 변경 시 raw에서 재빌드.

## 5. 대시보드 UI

- **첫 화면**: 대표 숫자 **1개**(3앱 누적 사용자) + 3앱 상태 카드(앱별 미니 스파크라인). 숫자 병렬 금지 — 서류작성_필수기준 §3(페이지당 대표 숫자 1개)을 대시보드에도 적용.
- **앱별 탭 3개**:
  - 구름한입: 푸시 캠페인 성과·리텐션 중심 (데이터마케터·그로스 인용 섹션)
  - 잡플로우: 유입 추이·카테고리 1위 스토리
  - 데일리픽: 스트릭·리텐션 메커닉. 데이터 적으면 숨기지 않고 "수집 시작일 ~" 정직 표기.
- **인사이트 노트 타임라인**: 날짜별 "관찰 → 해석 → 액션" 카드. 예: "리타겟 푸시 클릭 2.08% vs 신규 0.43% → 재방문 소구로 소재 재설계".
- 모바일 대응 필수(채용담당자가 폰으로 열 확률 높음). 한국어. `word-break: keep-all`.

## 6. 엣지·에러 처리

- 수집 누락 구간: 차트에 공백으로 정직하게 표시. 보간·날조 금지(검증된 수치만 원칙).
- 데이터 없는 지표: 섹션 유지 + "초기 계측 중" 명시.
- `build_dashboard.py`는 raw 스키마 변화에 **fail-loud** — 조용히 0으로 채우지 않고 에러로 중단.
